Cómo funciona

Getting Started

1

Explorar tareas abiertas

Los clientes publican tareas de desarrollo con recompensas. Navega por complejidad, lenguaje o monto de la recompensa.

2

Enviar un Pull Request

Escribe tu código y envía un pull request público. Varios desarrolladores pueden enviar PRs para la misma tarea.

3

Puntuación de complejidad

Tu PR se puntúa automáticamente según los archivos modificados, líneas agregadas, dificultad del lenguaje y cobertura de pruebas.

4

El cliente revisa y aprueba

El cliente revisa todos los PRs enviados y elige el mejor para aprobar. Al aprobar, se cobra automáticamente al cliente el monto de la recompensa.

5

Recibe tu pago

El desarrollador recibe el pago a través de su método preferido. Desbloquea insignias de logros y sube en la tabla de ganancias con cada PR aprobado.

Para clientes

  1. Inicia sesion con GitHub o Google y configura tu metodo de pago
  2. Crea una tarea con descripcion, enlace al repositorio y monto de recompensa
  3. Los desarrolladores envian pull requests publicos para tu tarea
  4. Revisa todos los pull requests enviados y aprueba el mejor
  5. Se te cobra automaticamente la recompensa y el desarrollador recibe el pago

Guia del flujo de trabajo

Sigue estos pasos para completar el flujo de trabajo completo desde el inicio de sesion hasta el pago.

1

Iniciar sesion via SSO

Autenticarse con Google, GitHub, Microsoft u Okta.

2

Verificacion de identidad KYC

3

Crear una tarea

Publicar una tarea con recompensa para que los desarrolladores la completen.

Crear tarea Inicio de sesion requerido
4

Subir codigo fuente

Detalle de tarea Inicio de sesion requerido

Explorar tareas abiertas
