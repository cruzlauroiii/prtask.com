Как это работает

Getting Started

1

Просмотрите открытые задачи

Клиенты публикуют задачи на разработку с вознаграждениями. Просматривайте по сложности, языку или сумме вознаграждения.

2

Отправьте Pull Request

Напишите код и отправьте публичный pull request. Несколько разработчиков могут отправить PR на одну задачу.

3

Оценка сложности

Ваш PR автоматически оценивается по изменённым файлам, добавленным строкам, сложности языка и покрытию тестами.

4

Клиент проверяет и одобряет

Клиент просматривает все отправленные PR и выбирает лучший для одобрения. При одобрении с клиента автоматически списывается сумма вознаграждения.

5

Получите оплату

Разработчик получает оплату через предпочтительный способ. Открывайте значки достижений и поднимайтесь в рейтинге заработка с каждым одобренным PR.

Для клиентов

  1. Войдите через GitHub или Google и настройте способ оплаты
  2. Создайте задачу с описанием, ссылкой на репозиторий и суммой награды
  3. Разработчики отправляют публичные пул-реквесты для вашей задачи
  4. Просмотрите все отправленные пул-реквесты и одобрите лучший
  5. С вас автоматически списывается награда, а разработчик получает оплату

Руководство по рабочему процессу

Следуйте этим шагам для завершения полного рабочего процесса от входа до оплаты.

1

Вход через SSO

Аутентификация через Google, GitHub, Microsoft или Okta.

Войти Обязательно
2

KYC-верификация

Пройдите верификацию личности для активации платежей и соблюдения нормативов.

Настройки - KYC Требуется вход
3

Создать задачу

Опубликуйте задачу с наградой для разработчиков.

Создать задачу Требуется вход
4

Загрузить исходный код

Загрузите папку с исходным кодом в задачу, чтобы разработчики видели diff.

Детали задачи Требуется вход
5

Просмотреть открытые задачи

Разработчики просматривают доступные задачи и находят работу, соответствующую их навыкам.

Задачи Открыто
6

Просмотр деталей задачи

Просмотрите описание задачи, сумму награды и существующие пул-реквесты.

Детали задачи Открыто
7

Редактировать код в diff-редакторе

Редактируйте исходный код прямо в diff-редакторе браузера.

Diff-редактор Требуется вход
8

Отправить пул-реквест

Отправьте изменения кода как пул-реквест для задачи.

Отправить PR Требуется вход
9

Просмотр изменений кода

Клиент просматривает diff, чтобы увидеть все изменения кода, отправленные разработчиком.

Детали PR Требуется вход
10

Добавить комментарии к ревью

Добавляйте инлайн-комментарии к конкретным строкам или выделенному коду.

Комментарии PR Требуется вход
11

Ответить и решить комментарии

Разработчик применяет исправления, отвечает на комментарии и решает их.

Решить комментарии Требуется вход
12

Одобрить PR и оплатить через Maya

Клиент одобряет PR и оплачивает награду через кредитную карту Maya.

Одобрить и оплатить Платежи Требуется вход
13

История платежей

Просмотр всех платёжных транзакций и их статусов.

Платежи Требуется вход
14

Настройки вывода

Настройте способ вывода и проверьте баланс для вывода.

Настройки Требуется вход
