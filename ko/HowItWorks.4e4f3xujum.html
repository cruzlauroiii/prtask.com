이용 방법

Getting Started

1

열린 작업 둘러보기

클라이언트가 보상이 있는 개발 작업을 게시합니다. 복잡도, 언어 또는 보상 금액별로 둘러보세요.

2

풀 리퀘스트 제출

코드를 작성하고 공개 풀 리퀘스트를 제출하세요. 여러 개발자가 같은 작업에 PR을 제출할 수 있습니다.

3

복잡도 점수

PR은 변경된 파일, 추가된 줄 수, 언어 난이도 및 테스트 커버리지를 기반으로 자동 점수가 매겨집니다.

4

클라이언트 검토 및 승인

클라이언트가 제출된 모든 PR을 검토하고 최적의 것을 승인합니다. 승인 시 클라이언트에게 보상 금액이 자동 청구됩니다.

5

보수 받기

개발자는 선호하는 지급 방법으로 보수를 받습니다. 승인된 PR마다 업적 배지를 해제하고 수익 리더보드에서 순위를 올리세요.

클라이언트용

  1. GitHub 또는 Google로 로그인하고 결제 방법을 설정
  2. 설명, 저장소 링크, 보상 금액이 포함된 작업을 생성
  3. 개발자가 작업에 대한 공개 풀 리퀘스트를 제출
  4. 제출된 모든 풀 리퀘스트를 검토하고 최선의 것을 승인
  5. 보상이 자동으로 청구되고 개발자에게 지불됩니다

워크플로우 가이드

로그인부터 결제까지 전체 워크플로우를 완료하려면 이 단계를 따르세요.

1

SSO로 로그인

Google, GitHub, Microsoft 또는 Okta로 인증합니다.

로그인 필수
2

KYC 본인 확인

결제를 활성화하고 규정을 준수하기 위해 본인 확인을 완료합니다.

설정 - KYC 로그인 필요
3

작업 생성

개발자가 완료할 보상이 있는 작업을 게시합니다.

작업 생성 로그인 필요
4

소스 코드 업로드

소스 코드 폴더를 작업에 업로드하여 개발자가 diff를 볼 수 있도록 합니다.

작업 상세 로그인 필요
5

오픈 작업 탐색

개발자가 사용 가능한 작업을 탐색하고 자신의 기술에 맞는 일을 찾습니다.

작업 오픈
6

작업 상세 보기

작업 설명, 보상 금액, 기존 풀 리퀘스트를 확인합니다.

작업 상세 오픈
7

Diff 편집기에서 코드 수정

브라우저의 diff 편집기에서 소스 코드를 직접 편집합니다.

Diff 편집기 로그인 필요
8

풀 리퀘스트 제출

코드 변경사항을 작업의 풀 리퀘스트로 제출합니다.

PR 제출 로그인 필요
9

코드 변경사항 검토

클라이언트가 diff를 검토하여 개발자가 제출한 모든 코드 변경사항을 확인합니다.

PR 상세 로그인 필요
10

리뷰 코멘트 추가

변경이 필요한 특정 줄이나 하이라이트된 코드에 인라인 코멘트를 추가합니다.

PR 코멘트 로그인 필요
11

코멘트 답변 및 해결

개발자가 수정을 적용하고 코멘트에 답변하고 해결합니다.

코멘트 해결 로그인 필요
12

PR 승인 및 Maya로 결제

클라이언트가 PR을 승인하고 Maya 신용카드 결제로 보상을 지불합니다.

승인 및 결제 결제 로그인 필요
13

결제 내역

모든 결제 거래와 상태를 확인합니다.

결제 로그인 필요
14

출금 설정

출금 방법을 설정하고 출금 잔액을 확인합니다.

설정 로그인 필요
