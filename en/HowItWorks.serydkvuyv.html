How It Works

Getting Started

1

Browse Open Tasks

Clients post development tasks with bounties. Browse by complexity, language, or reward amount.

2

Submit a Pull Request

Write your code and submit a public pull request. Multiple developers can submit PRs for the same task.

3

Complexity Scoring

Your PR is automatically scored based on files changed, lines added, language difficulty, and test coverage.

4

Client Reviews and Approves

The client reviews all submitted PRs and chooses the best one to approve. On approval, the client is automatically charged the bounty amount.

5

Get Paid

The developer receives payment via their preferred payout method. Unlock achievement badges and climb the earnings leaderboard with every approved PR.

For Clients

  1. Sign in with GitHub or Google and set up your payment method
  2. Create a task with a description, repository link, and bounty amount
  3. Developers submit public pull requests for your task
  4. Review all submitted pull requests and approve the best one
  5. You are automatically charged the bounty and the developer gets paid
