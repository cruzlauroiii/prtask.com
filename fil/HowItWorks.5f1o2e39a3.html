Paano Gumagana

Pagsisimula

1

Mag-browse ng Mga Bukas na Gawain

Nagpo-post ang mga kliyente ng mga gawain sa development na may mga bounty. Mag-browse ayon sa pagiging kumplikado, wika, o halaga ng reward.

2

Magsumite ng Pull Request

Isulat ang iyong code at magsumite ng pampublikong pull request. Maraming developer ang maaaring magsumite ng mga PR para sa parehong gawain.

3

Complexity Scoring

Awtomatikong sino-score ang iyong PR batay sa mga file na binago, mga linya na idinagdag, kahirapan ng wika, at test coverage.

4

Nire-review at Ina-approve ng Kliyente

Rere-review ng kliyente ang lahat ng naisumiteng PR at pipiliin ang pinakamahusay na i-approve. Sa pag-approve, awtomatikong sinisingil ang kliyente ng halaga ng bounty.

5

Mabayaran

Natatanggap ng developer ang bayad sa pamamagitan ng kanilang gustong paraan ng payout. Mag-unlock ng mga achievement badge at umakyat sa earnings leaderboard sa bawat na-approve na PR.

Para sa mga Kliyente

  1. Mag-sign in gamit ang GitHub o Google at i-setup ang iyong paraan ng bayad
  2. Gumawa ng gawain na may paglalarawan, link ng repositoryo, at halaga ng bounty
  3. Nagsusumite ang mga developer ng mga pampublikong pull request para sa iyong gawain
  4. I-review ang lahat ng naisumiteng pull request at i-approve ang pinakamahusay
  5. Awtomatiko kang sinisingil ng bounty at mababayaran ang developer

Gabay sa Workflow

Sundin ang mga hakbang na ito upang makumpleto ang buong workflow mula sa pag-login hanggang sa bayad.

1

Mag-sign In sa pamamagitan ng SSO

Mag-authenticate gamit ang Google, GitHub, Microsoft, o Okta.

Mag-sign In Kinakailangan
2

KYC Pag-verify ng Pagkakakilanlan

Kumpletuhin ang pag-verify ng pagkakakilanlan upang i-enable ang mga bayad at sumunod sa mga regulasyon.

Settings - KYC Kinakailangan ang Pag-login
3

Gumawa ng Gawain

Mag-post ng gawain na may bounty para kumpletuhin ng mga developer.

Gumawa ng Gawain Kinakailangan ang Pag-login
4

Mag-upload ng Source Code

I-upload ang source code folder sa gawain upang makita ng mga developer ang diff.

Detalye ng Gawain Kinakailangan ang Pag-login
5

Mag-browse ng mga Bukas na Gawain

Nag-browse ang mga developer ng mga available na gawain at naghahanap ng trabahong tugma sa kanilang mga kasanayan.

Mga Gawain Bukas
6

Tingnan ang Detalye ng Gawain

Tingnan ang paglalarawan ng gawain, halaga ng bounty, at mga umiiral na pull request.

Detalye ng Gawain Bukas
7

Baguhin ang Code sa Diff Editor

I-edit ang source code direkta sa browser diff editor.

Diff Editor Kinakailangan ang Pag-login
8

Isumite ang Pull Request

Isumite ang iyong mga pagbabago sa code bilang pull request para sa gawain.

Isumite ang PR Kinakailangan ang Pag-login
9

I-review ang mga Pagbabago sa Code

Rere-review ng kliyente ang diff upang makita ang lahat ng pagbabago sa code na isinumite ng developer.

Detalye ng PR Kinakailangan ang Pag-login
10

Magdagdag ng mga Komento sa Review

Magdagdag ng mga inline na komento sa mga partikular na linya o naka-highlight na code na nangangailangan ng pagbabago.

Mga Komento sa PR Kinakailangan ang Pag-login
11

Sumagot at Resolbahin ang mga Komento

Nag-a-apply ng mga ayos ang developer, sumasagot sa mga komento, at nire-resolve ang mga ito.

Resolbahin ang mga Komento Kinakailangan ang Pag-login
12

I-approve ang PR at Magbayad sa pamamagitan ng Maya

Ina-approve ng kliyente ang PR at binabayaran ang bounty sa pamamagitan ng Maya credit card checkout.

I-approve at Magbayad Mga Bayad Kinakailangan ang Pag-login
13

Kasaysayan ng Bayad

Tingnan ang lahat ng mga transaksyon sa bayad at ang kanilang katayuan.

Mga Bayad Kinakailangan ang Pag-login
14

Mga Setting ng Withdrawal
