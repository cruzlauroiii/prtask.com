No platform fees — developers receive 100% of the bounty.

Create a Task — Post Bounties for Development Work

The Create Task page is where clients post new development work to the PrTask marketplace. Each task requires a clear title, detailed description, linked GitHub repository URL, and a bounty amount that developers will earn upon successful completion. The task creation form guides you through setting all the parameters that help developers understand what needs to be done, how complex the work is, and what they will earn for completing it. Well-defined tasks attract more qualified developers and lead to faster, higher-quality submissions.

Complexity levels (Trivial, Simple, Medium, Complex, Expert) help developers gauge the expected effort and skill requirements before committing to a task. Set the complexity honestly to attract the right developers — underestimating complexity may result in incomplete submissions, while overestimating may deter developers from attempting the task. The bounty amount should reflect the complexity and value of the work. PrTask enforces a minimum bounty to ensure meaningful compensation, and developers see the full bounty amount with no hidden deductions. Tags help categorize your task by technology and type for better discoverability.

You can also set work item types (Epic, Feature, Story, Task, Bug) to align with your project management methodology, and choose between public and private visibility. Public tasks are visible to all developers on the marketplace, while private tasks are only accessible to invited developers. After submitting a task, it becomes immediately available on the Tasks page where developers can discover and start working on it. Payment for the bounty is processed when you approve and merge the winning pull request, ensuring you only pay for work that meets your requirements and passes your code review.
