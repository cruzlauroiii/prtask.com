运作方式

Getting Started

1

浏览开放任务

客户发布带有赏金的开发任务。按复杂度、语言或奖励金额浏览。

2

提交拉取请求

编写代码并提交公开的拉取请求。多个开发者可以为同一任务提交 PR。

3

复杂度评分

您的 PR 将根据更改的文件、添加的行数、语言难度和测试覆盖率自动评分。

4

客户审查和批准

客户审查所有提交的 PR 并选择最佳方案进行批准。批准后，系统自动向客户收取赏金金额。

5

获得报酬

开发者通过首选支付方式收到报酬。每个获批的 PR 都可解锁成就徽章，并在收入排行榜上攀升。

面向客户

  1. 使用 GitHub 或 Google 登录并设置支付方式
  2. 创建包含描述、仓库链接和赏金金额的任务
  3. 开发者为您的任务提交公开的拉取请求
  4. 审查所有提交的拉取请求并批准最佳方案
  5. 系统自动向您收取赏金，开发者获得报酬

工作流程指南

按照这些步骤完成从登录到支付的完整工作流程。

1

通过 SSO 登录

通过 Google、GitHub、Microsoft 或 Okta 进行身份验证。

登录 必需
2

KYC 身份验证

完成身份验证以启用支付并遵守法规。

设置 - KYC 需要登录
3

创建任务

发布带赏金的任务供开发者完成。

创建任务 需要登录
4

上传源代码

将源代码文件夹上传到任务，以便开发者查看差异。

任务详情 需要登录
5

浏览开放任务

开发者浏览可用任务，找到与其技能匹配的工作。

任务 开放
6

查看任务详情

查看任务描述、赏金金额和现有拉取请求。

任务详情 开放
7

在差异编辑器中修改代码

直接在浏览器差异编辑器中编辑源代码。

差异编辑器 需要登录
8

提交拉取请求

将您的代码更改作为拉取请求提交到任务。

提交 PR 需要登录
9

审查代码更改

客户审查差异以查看开发者提交的所有代码更改。

PR 详情 需要登录
10

添加审查评论

在需要修改的特定行或高亮代码上添加内联评论。

PR 评论 需要登录
11

回复并解决评论

开发者应用修复，回复评论并解决它们。

解决评论 需要登录
12

批准 PR 并通过 Maya 支付

客户批准 PR 并通过 Maya 信用卡结账支付赏金。

批准并支付 付款 需要登录
13

支付历史

查看所有支付交易及其状态。

付款 需要登录
14

提现设置

配置提现方式并验证提现余额。

设置 需要登录
