Repository Management — Connect and Configure Your GitHub Repos

The Repository page allows clients to connect their GitHub repositories to PrTask for seamless task management and pull request workflows. By linking a repository, you enable developers to submit pull requests directly against your codebase when working on tasks you have posted. Each connected repository can have a custom display name for easy identification within PrTask, and visibility settings control whether the repository is publicly listed or restricted to specific developers.

Managing your connected repositories is essential for maintaining an organized development workflow on PrTask. When you create a task, you link it to one of your connected repositories so developers know exactly where their code contributions should be submitted. Public repositories are visible to all marketplace participants, making them ideal for open source projects and community-driven development. Private repositories restrict access to invited developers only, suitable for proprietary codebases where you want to control who can view and contribute to your code.

PrTask integrates with GitHub through standard API connections, requiring only the repository URL to establish the link. You maintain full control over your repositories — PrTask does not modify repository settings, branch protections, or access permissions. All pull request reviews and merges are performed through your existing GitHub workflow, with PrTask handling the bounty payment and task status tracking automatically. As you add more repositories, the Repository page serves as your central directory for all codebases connected to the PrTask marketplace.
